# 开发提案

## 项目概述

趣享社(QuXiangShe)社交平台后端项目是基于SpringBoot3.2构建的高并发社交内容平台，支持用户发布动态、浏览Feed流、评论互动、AI内容摘要等核心功能。项目面向高并发场景设计，采用推拉结合的Feed流、无限层级评论、Redis滑动窗口限流等技术方案。

## 开发目标

1. **实现完整的用户认证体系**：JWT双令牌认证，支持注册登录、令牌刷新、登出
2. **构建高性能Feed流系统**：推拉结合模式，支持游标分页无限滚动
3. **实现无限层级评论系统**：单表存储，两层扁平展示，异步写入
4. **集成Spring AI内容摘要**：动态发布后自动生成AI摘要
5. **实现高并发限流**：Redis滑动窗口限流，支持每秒100次请求限制

## 技术架构

### 技术栈

| 层级 | 技术选型 |
|------|----------|
| 框架 | Spring Boot 3.2.x |
| 安全 | Spring Security 6.2.x + JWT |
| AI | Spring AI 1.0.x |
| 数据库 | MySQL 8.0.x |
| 缓存 | Redis 7.0.x |
| 消息队列 | RabbitMQ 3.12.x |
| 搜索引擎 | Elasticsearch 8.0.x |
| 工具 | Lombok |

### 核心技术方案

1. **JWT双令牌认证**：AccessToken(15分钟)+RefreshToken(7天)
2. **Feed流推拉结合**：Push模式(粉丝数<1000)+Pull模式兜底
3. **游标分页**：基于lastPostTime+lastPostId的游标分页
4. **无限层级评论**：parent_id自关联+时间分桶+异步写入
5. **Redis滑动窗口限流**：ZSet实现滑动窗口限流

## 验收标准

### 功能验收

| 功能点 | 验收条件 |
|--------|----------|
| 用户注册 | 可正常注册，返回用户ID |
| 用户登录 | 返回AccessToken+RefreshToken |
| 令牌刷新 | AccessToken过期后可用RefreshToken刷新 |
| 发布动态 | 动态保存成功，触发AI摘要任务 |
| Feed流拉取 | 游标分页返回动态，支持无限滚动 |
| 评论发布 | 评论写入Redis队列，5秒内落库 |
| 评论查询 | 返回顶级评论+二级子评论 |
| AI摘要 | 动态发布后生成摘要 |
| 限流 | 超过100次/秒返回429 |

### 性能验收

| 指标 | 目标值 |
|------|--------|
| Feed流P99 | < 200ms |
| 评论列表P99 | < 300ms |
| QPS支撑 | 5000+ |

### 代码验收

- [ ] Service层单元测试覆盖率 > 70%
- [ ] 代码无编译警告
- [ ] 符合code_style.md规范

## 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| AI服务不可用 | 摘要生成失败 | 降级展示原文前100字 |
| Redis故障 | 限流/收件箱不可用 | 降级到MySQL兜底 |
| 高并发Feed拉取 | 数据库压力 | 推模式优先，推送失败兜底拉取 |
