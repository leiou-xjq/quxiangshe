## 核心技术方案

### 1. JWT双令牌认证

- AccessToken：15分钟有效期，Payload含userId、username、roles
- RefreshToken：7天有效期，存储于Redis，Key为`refresh:{userId}`
- 令牌刷新：检查Redis中RefreshToken存在性，验证通过后生成新AccessToken
- 登出操作：删除Redis中的RefreshToken

### 2. Feed流推拉结合

| 策略 | 触发条件 | 数据源 | 适用场景 |
|------|----------|--------|----------|
| 推模式(Push) | 粉丝数<1000 | Redis List inbox:{userId} | 活跃用户 |
| 拉模式(Pull) | 收件箱为空/过期 | MySQL feed表 | 新用户/低活跃 |
| 混合模式 | 同时存在 | 合并Redis+MySQL结果 | 通用 |

### 3. 无限层级评论

- 存储：单表存储，parent_id自关联
- 查询：两层扁平展示(顶级+二级子评论)
- 展示：前端递归渲染完整评论树
- 分桶：按小时分桶(time_bucket = UNIX_TIMESTAMP(created_at)/3600)

### 4. Redis滑动窗口限流

```java
// ZSet实现滑动窗口
String key = "rate_limit:" + userId + ":" + currentSecond;
redis.zadd(key, currentTimestamp, requestId);
redis.zremrangeByScore(key, 0, currentTimestamp - windowSize * 1000);
long count = redis.zcard(key);
```

### 5. 异步消息队列

- RabbitMQ Exchange配置：fanout.exchange(Feed扩散)、direct.exchange(AI任务)
- Queue命名：feed.push.queue、ai.summary.queue
- 消息持久化：deliveryMode=2
- 消费者确认：autoAck=false，手动确认
