# 需求文档

## 项目背景

趣享社(QuXiangShe)是一款面向年轻群体的社交内容分享平台，支持用户发布图文动态、浏览Feed流、参与评论互动，并集成AI内容摘要功能提升阅读体验。项目采用微服务架构设计，面向高并发场景，确保海量用户同时在线时的系统稳定性。

## 核心功能

### 1. 用户认证模块

- **双令牌JWT认证**：AccessToken(15分钟)+RefreshToken(7天)
- **注册/登录**：手机号/邮箱注册，密码加密存储
- **令牌刷新**：RefreshToken自动换发AccessToken
- **登出**：主动失效RefreshToken

### 2. Feed流模块

- **分层推拉模式**：
  - 推模式(Push)：用户发布动态时，异步写入粉丝的收件箱(Redis List)
  - 拉模式(Pull)：用户首次登录或收件箱为空时，从关注者动态表拉取
- **游标分页**：基于时间戳的游标分页，避免传统分页偏移量性能问题
- **无限滚动**：前端传递lastPostTime+lastPostId实现无限滚动

### 3. 评论系统

- **无限层级存储**：评论以parent_id实现无限嵌套，数据库单表存储
- **两层扁平展示**：API返回顶级评论+二级直接子评论，内部递归组装完整树
- **时间分桶策略**：按created_at小时分桶，查询时按分桶ID定位，减少全表扫描
- **异步写入**：评论创建先写Redis队列，批量异步落库MySQL

### 4. AI内容摘要

- **自动摘要**：用户发布动态后，Spring AI调用LLM生成内容摘要
- **存储优化**：摘要异步写入，存入ai_summary表与Post关联
- **展示降级**：AI服务不可用时展示原文前100字

### 5. 高并发防护

- **Redis滑动窗口限流**：基于ZSet实现滑动窗口，每秒100次请求限制
- **请求去重**：同一token请求5秒内去重
- **熔断降级**：Redis不可用时降级为数据库限流

## 非功能需求

### 高并发指标

- Feed流接口P99响应时间 < 200ms
- 评论列表加载P99 < 300ms
- 系统支持10万同时在线用户
- 高峰期QPS支撑5000+

### 可用性

- 系统可用性目标 99.9%
- 核心接口月度不可用时间 < 43分钟

### 扩展性

- Feed流采用推拉结合，分库分表设计预留
- 评论系统支持无限层级，水平扩展无忧
