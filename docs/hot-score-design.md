# 热点笔记排序设计文档

## 1. 概述

热点笔记排序系统负责在全站范围内筛选和排序高热度笔记，为用户提供热门内容推荐。本系统采用**时间分桶 + 热度分数**的方案，支持高效查询和自然衰减。

## 2. 当前实现

### 2.1 HotScoreScheduler

现有实现采用**定时任务 + Redis ZSet**的方式：

| 特性 | 说明 |
|------|------|
| 存储结构 | 单一 Redis ZSet (`note:hot`) |
| 更新方式 | 每小时增量同步 + 每日衰减 |
| 热度分数 | `likeCount×1 + commentCount×2 + favoriteCount×3 + forwardCount×5` |

**问题**：
- 无分桶机制，所有笔记存在一个 ZSet，量大时查询效率下降
- 热度衰减依赖定时任务，无法实时反映
- 无热点筛选，所有笔记统一排序

## 3. 设计方案：时间分桶 + 聚合

### 3.1 分桶策略

**按日期分桶，滚动30天窗口**：

| Key Pattern | 说明 |
|------------|------|
| `note:bucket:{yyyyMMdd}:all` | 当日全部笔记桶（按时间排序） |
| `note:bucket:{yyyyMMdd}:hot` | 当日热点笔记桶（热度分数 >= 阈值） |
| `note:hot` | 全局热点池（来自各桶的热点汇总） |

**TTL**：每个桶存储 30 天，到期自动过期。

### 3.2 写入流程

```
新笔记发布
    ↓
计算热度分数
    ↓
写入当日时间桶 (note:bucket:{date}:all)
    ↓
分数 >= 热点阈值 (如 50 分)?
    ↓是               ↓否
写入热点桶          仅在时间桶
写入全局热点池
    ↓
定时更新分数
```

### 3.3 查询流程

**获取热点笔记列表**：

```
请求热点笔记 N 条
    ↓
查 note:hot 全局池，ZREVRANGE 取前 N 条
    ↓
已够 N 条? ──是──→ 直接返回
    ↓否
跨最近 30 天桶聚合补充
    ↓
按热度分数去重、返回
```

**游标分页**：

```
第一页：ZREVRANGE note:hot 0 9
第二页：以第10条笔记的分数为游标
       ZREVRANGE note:hot 10 19
```

## 4. 热度分数计算

### 4.1 分数公式

```
热度分数 = likeCount × 1.0 + commentCount × 2.0 + favoriteCount × 3.0 + forwardCount × 5.0
```

### 4.2 权重说明

| 行为 | 权重 | 说明 |
|------|------|------|
| 点赞 | × 1 | 轻度互动 |
| 评论 | × 2 | 中度互动 |
| 收藏 | × 3 | 深度认可 |
| 转发 | × 5 | 主动传播 |

### 4.3 热点阈值

| 阈值 | 效果 |
|------|------|
| >= 50 | 进入当日热点桶 |
| >= 100 | 进入全局热点池 |

## 5. 热度衰减

### 5.1 衰减策略

**每日凌晨执行衰减任务**：

```
遍历 note:hot 全局池
    ↓
所有分数 × 0.9 衰减系数
    ↓
分数 < 阈值?
    ↓是
移出热点池
    ↓
旧桶自然过期（30天 TTL）
```

### 5.2 为什么要衰减

- 新笔记有更多曝光机会
- 旧笔记热度自然下降
- 保证热门榜的新鲜度

## 6. 与评论排序的对比

| 维度 | 评论 BucketSortStrategy | 热点笔记排序（设计） |
|------|----------------------|-------------------|
| 分桶键 | `post:{id}:bucket:{date}:root_comments` | `note:bucket:{date}:all` |
| 热点键 | `post:{id}:hot:root_comments` | `note:hot` |
| 分数公式 | 点赞×2 + 回复×3 - 时间衰减 | 点赞×1 + 评论×2 + 收藏×3 + 转发×5 |
| 热点判定 | 分数超过阈值 | 分数超过阈值 |
| TTL | 30天 | 30天 |
| 衰减方式 | 查询时实时衰减 | 定时任务批量衰减 |

## 7. Redis 数据结构

### 7.1 ZSet 用法

```bash
# 添加笔记到热点池（分数即热度）
ZADD note:hot 125.0 "note_12345"

# 获取热度前10的笔记
ZREVRANGE note:hot 0 9 WITHSCORES

# 更新笔记热度分数
ZINCRBY note:hot 10.0 "note_12345"

# 衰减：所有分数乘以0.9
ZRANGEBYSCORE note:hot 0 +inf WITHSCORES
# 然后重新 ZADD
```

### 7.2 Key 设计

```
note:hot                          # 全局热点池 ZSet
note:bucket:20260512:all          # 2026-05-12 全部笔记桶 ZSet
note:bucket:20260512:hot          # 2026-05-12 热点笔记桶 ZSet
note:meta:{noteId}                # 笔记元数据（用于分桶定位）
```

## 8. 性能优化

### 8.1 为什么分桶能提升性能

| 场景 | 优化前 | 优化后 |
|------|-------|-------|
| 查询热点 | 全量扫描 note:hot | 先查 note:hot，不够再跨桶 |
| 更新分数 | 只能更新全局池 | 可按桶隔离更新 |
| 数据过期 | 无机制 | TTL 自动滚动 |

### 8.2 分桶优势

1. **查询高效**：热点笔记从专用池获取，避免全量扫描
2. **更新隔离**：单个桶更新不影响其他桶
3. **自然过期**：旧桶自动删除，无需清理任务
4. **可扩展**：可按标签、分类等维度增加分桶
