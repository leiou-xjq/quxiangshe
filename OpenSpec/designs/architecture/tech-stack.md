# 技术选型

## 概述

本文档描述系统的技术栈选择

## 前端技术栈

| 技术 | 版本 | 说明 |
-----|------|------|
| React | 18.x | UI框架 |
| TypeScript | 5.x | 类型系统 |
| Vite | 5.x | 构建工具 |
| Zustand | 4.x | 状态管理 |
| React Query | 5.x | 数据获取 |
| Ant Design | 5.x | 组件库 |

## 后端技术栈

| 技术 | 版本 | 说明 |
-----|------|------|
| Java | 17 | 运行环境 |
| Spring Boot | 3.x | 框架 |
| Spring Cloud | 2023.x | 微服务 |
| MyBatis Plus | 3.x | ORM框架 |

## 数据存储

| 技术 | 版本 | 说明 |
-----|------|------|
| MySQL | 8.0 | 关系型数据库 |
| Redis | 7.x | 缓存 |
| Elasticsearch | 8.x | 搜索 |

## 基础设施

| 技术 | 说明 |
-----|------|
| Docker | 容器化 |
| Kubernetes | 容器编排 |
| Jenkins | CI/CD |
| SkyWalking | 链路追踪 |
| ELK | 日志分析 |

## 选择理由

### 前端

- React：生态丰富，团队熟悉
- TypeScript：类型安全
- Vite：开发体验好
- Ant Design：组件丰富

### 后端

- Spring Boot：生态完善
- Java 17：性能优秀
- MyBatis Plus：开发效率高

### 数据

- MySQL：成熟稳定
- Redis：性能优秀
- Elasticsearch：搜索能力
